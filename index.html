  • Metaprogramming-based regular expressions in C++20

    Overview

      This project involves writing a custom regular expression parser and matcher in C++20. What makes this approach special is that the engine is capable of compiling the regular expression into native C++ code, making it much more efficient than standard runtime-interpreted regular expressions. It is also possible to use this kind of regex in a constexpr context, allowing for compile-time matching in cases where the input is a constant expression. This is, of course, achieved through heavy template metaprogramming, inspired by other similar libraries like CTRE or Boost.Xpressive. Basically, this approach makes the regular expression a compile-time known data structure. Instead of creating a std::regex object, which is painfully slow in C++, it is enough to specify a typedef with the C++ using directive, which will alias a particular regular expression. For example, one can write using pattern = meta::regex<"ab*"> to generate the code for matching the regular expression ab*. The generated type contains static methods for performing commong regex operations like matching, searching or iterating through a set of results.

    Private (for now)

    Matching

      Matching is implemented using the backtracking approach, since this algorithm is capable of matching complex syntactical structures like backreferences and capturing groups. The actual implementation is a bit tricky and uses Continuation-Passing techniques to create the necessary control flow, because matching certain elements requires a lot of future context (for example, matching (a|ab)+b), which is difficult to directly implement starting from the Abstract Syntax Tree of the regular expression. The main difficulty comes from the fact that it is impossible to dynamically allocate memory in a constexpr function and use it later, at runtime.

    The matching API supports ranges as well as iterator pairs. Any container which satisfies the meta::char_range concept can be matched. That is, it must be a std::ranges::forward_range and its value type must be convertible to char32_t. Matching will return an instance of meta::regex_result, a type which contains information about whether the match succeeded and the captured content (including capturing groups, if present). This type is also explicitly convertible to bool. The library currently supports a lot of the basic PCRE syntax, including capturing groups, backreferences, sets and lookaheads.

  • Genetic Neural Network Trainer

    Overview

      This is an educational project aimed at understanding how neural networks work on a low level, as well as implementing a Genetic Algorithm for training a Multilayer-Perceptron neural network. The project is implemented in C++ and provides a small library abbreviated as gnnt. It contains a custom implementation of a neural network, including activation functions, utilities for dataset manipulation and training metrics. The neural network architecture is defined at compile-time for optimization purposes, using templates to specify the layers with their respective activation functions. The library was designed with genericity and efficiency in mind, supporting custom underlying data types instead of float or double. The implementation supports a variable number of hidden layers, each being specified as a template argument using the gnnt::mlp::layer class template. This has several performance advantages, like the ability of the compiler to better optimize the code, not needing to dynamically allocate memory at runtime and the possibility to implement forward propagation using a fold-expression (see the fold() method here).

    The main focus of the project was specifically on using a Genetic Algorithm to train the network. That is, each chromosome was represented by the weights and biases of the network, other parameters being taken as constants. As an example of a practical application, the project provides a desktop interface written in QT which uses the library to train a network. The graphical interface provides a 28x28 canvas where the user can draw grayscale images. The ultimate goal is, of course, to recognize the digits drawn using the trained neural network.

    Digit Recognition

      To train the network, the handwritten digits from the MNIST dataset have been used. A simple network with two hidden layers of 16 neurons each was able to achieve more than 95% accuracy on the test dataset. Although it is possible, in theory, to further improve this result, the training process was very long due to the size of the chromosomes. It's like searching a needle in a 13000-dimensional haystack.

    The full source code, as well as the used dataset and serialized models, can be found on GitHub and has the following project structure:

    • data/ - various data used for testing
      • data/mnist/ - dataset for handwritten digit recognition
      • data/models/ - models saved after training
    • gnnt/ - main library implementation
      • gnnt/gnnt/mlp/ - compile-time Multilayer Perceptron network
      • gnnt/gnnt/training/- genetic algorithm implementation
    • ui/ - user interface implemented using QT
    • test/ - test cases for activation functions with the Catch2 framework
